# CLAUDE.md

Project orientation for Claude (or any agent / new contributor) working
on Velour. **Read this first** before exploring the tree.

## What Velour is

Velour is a Django "meta-app" — an app factory that also happens to be
the app it generates. The repo is a single Django project (`velour/`)
with many `INSTALLED_APPS`, each one a feature of the system. New apps
land in the tree directly at the repo root (not via the default
`startapp` placement), and their templates live under the project-level
`templates/<appname>/` directory rather than the app's own templates dir.

## Running locally

```
venv/bin/python manage.py runserver 7777
```

Default dev port is 7777. SQLite database at `db.sqlite3` (gitignored).
A fresh checkout needs `migrate` and `createsuperuser` before it's
useful.

Production runs as user `swibliq` on snel.com / lucdh.nl, behind nginx
+ supervisor + gunicorn. Deploy artifacts (gunicorn, supervisor, nginx,
setup.sh, hotswap.sh) are generated from templates in
`app_factory/templates/deploy/` via:

```
python manage.py generate_deploy
```

## App layout conventions

Each app is a single directory at the repo root:

```
appname/
├── __init__.py
├── apps.py
├── admin.py
├── models.py
├── views.py
├── urls.py
└── migrations/
    └── __init__.py
```

Templates live under `templates/appname/` at the project root, not
inside the app dir. There is a single `templates/base.html` that every
page extends. Static files: source CSS in `static/css/style.css`, JS in
`static/js/`. Collectstatic output (`staticfiles/`) is gitignored.

## The secret-file protocol

Three secret files live at `BASE_DIR`, all chmod 600, all gitignored:

- `secret_key.txt` — Django `SECRET_KEY`. `settings.py` reads it if
  present, falls back to env var, then to insecure dev default.
- `health_token.txt` — bearer token for `/sysinfo/health.json`.
  Generated by `python manage.py init_health_token`.
- `mail_relay_token.txt` — bearer token for the HTTP mail relay
  endpoint. Generated by `python manage.py init_mail_relay_token`.

`adminsetup.sh` and `hotswap.sh` rsync-exclude these so dev secrets
never leak to prod and prod secrets never get clobbered.

**Never commit secrets.** `.gitignore` has explicit entries for the
three above plus `*.token`. New secret-bearing files must be added to
`.gitignore` *before* staging.

## Apps currently in INSTALLED_APPS

(see `velour/settings.py` for the authoritative list)

- **dashboard** — landing/home page, nav routing.
- **terminal** — web terminal (channels / WebSocket).
- **app_factory** — creates new Django projects from templates.
- **sysinfo** — load/memory/disk/processes; hosts `/sysinfo/health.json`.
- **agricola** — board game (unrelated to the lab; pre-existing).
- **graphs** — Chart.js-based system graphs.
- **services** — systemd service management UI.
- **logs** — system log viewer (syslog, auth, nginx, supervisor, dmesg, journalctl).
- **identity** — singleton `Identity` model: name, mood, journal,
  hostname (used by deploy artifact generation).
- **security** — security UI.
- **landingpage** — newspaper-style chronicle landing page.
- **winctl** — Windows (UI) control via WSL.
- **maintenance** — backup / restore.
- **hosts** — `RemoteHost` model. Polls OTHER velour instances'
  health endpoints.
- **mailboxes** — outgoing mail. `MailAccount` (SMTP+IMAP) +
  `DynamicMailboxBackend` set as `EMAIL_BACKEND`.
- **mailroom** — inbound mail. `InboundMessage` model + IMAP polling.
- **experiments** — minimal `Experiment` model; will grow.
- **nodes** — physical microcontroller fleet (ESP8266 / ESP32 / Pico).
  `HardwareProfile` + `Node` + `SensorReading` models. Machine-facing
  API at `/api/nodes/<slug>/report/` with bearer token auth.
- **chronos** — time + timezones. Singleton `ClockPrefs` +
  `WatchedTimezone`. Topbar clock context processor. JS ticker with
  periodic re-sync against the server.
- **databases** — MySQL / PostgreSQL connection registry. Phase 1
  stores credentials and tests connections via pymysql / psycopg.
  Phase 2 will add table browsing and a SQL shell.

## House style

- Comments only when the *why* is non-obvious. Don't restate what the
  code does. Don't reference task/PR numbers in code comments.
- Edit existing files; don't add new ones unless the task requires it.
- Keep abstractions concrete. Three similar lines beats a premature
  helper.
- UI is quietly Tufte-influenced: small specific labels, high
  data-ink ratio, minimal chrome, color used sparingly and
  meaningfully.

## Things to *not* do

- **Don't `git add -A` blindly.** Review the diff first. `.gitignore`
  is strong but won't catch secrets pasted into source files.
- **Don't commit `db.sqlite3`.** It contains live tokens and data.
- **Don't switch the GitHub remote from HTTPS to SSH.** Auth is set
  up for HTTPS + a fine-grained PAT scoped to this single repo.
- **Don't rename Django apps casually.** Migration tables are keyed
  by app label.
- **Don't bypass git hooks** (`--no-verify`, `--no-gpg-sign`) without
  an explicit ask.
