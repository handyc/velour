# HexNN — ESP32-S3 SuperMini + ST7735 128×128

The full /hexnn/ pipeline on a stick. Generates a random K-dialable
nearest-neighbour genome, optionally runs a Hunt + Refine GA, then
runs the elite forever — same algorithm as the in-browser HexNN
bench, byte-for-byte at the same seed.

What's added vs. `hexnn_search/esp32_s3/`:

- **ST7735 1.44" panel** (128×128, `INITR_144GREENTAB`) wired to the
  same SPI pins the rest of the supermini lab uses
  (`SCK=12 MOSI=11 DC=4 CS=5 RST=6 BL=7`).
- **Diff-render** so SPI traffic is bounded — only the cells whose
  output changed since the last tick get repainted.
- **Initial full-screen render** at boot after the GA settles.

## Build + flash

```
pio run -t upload
pio device monitor
```

If your panel is a different ST7735 variant, change `INITR_144GREENTAB`
in `setup()` to the correct one (`INITR_BLACKTAB`, `INITR_GREENTAB`, etc.).

## Browser emulation

`/hexnn/tft/` on Velour runs the exact same algorithm in the browser
and renders to a 128×128 canvas with the same hex layout. Useful for
sanity-checking the output before flashing — same seed, same boot
genome, same step semantics.

## Generated by

`condenser.distill_hexnn.distill_hexnn_esp32s3(with_tft=True)` — see
the `_hexnn_tft_blocks()` helper for what's added vs. the no-TFT
variant. The non-TFT distillation at `hexnn_search/esp32_s3/` is
the canonical zero-display version.
