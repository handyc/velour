# cellular — Python port of s3lab Cellular sublab

Phase 4 of the cellular multi-platform port. Single-file Python 3,
stdlib only (no numpy, no Cython), drops cleanly into Velour's
existing Python ecosystem so taxon and condenser can simulate the
cellular kernel server-side.

## Run

```
./cellular.py                       # 200 rounds, derived seed
./cellular.py -r 100 -s 42 -p 25    # 100 rounds, seed 42, render every 25
./cellular.py --quiet               # final stats only, no per-render
```

## Performance

Pure Python is ~30× slower than the C distillation:

| port      | rounds/s |
|-----------|----------|
| C99       | ~350     |
| Python    | ~11      |
| OCaml     | ~50      |
| ESP32-S3  | ~15      |

Python's strength here isn't speed — it's *importability*. Velour
modules can `from cellular import step_grid, fitness, mutate` and
use the same kernel for slot pre-flight validation, taxon's
metric registry, or condenser's IR introspection.

## Algorithm parity

PRNG (xorshift32 + Park-Miller LCG), step_grid, fitness, mutate,
palette_inherit, neighbour_idx — all line-for-line ports of the
JS reference in `static/s3lab/js/engine.mjs`. Same hex-offset
neighbour tables.

What's different: `bytearray` instead of `Uint8Array`, `dataclass`
for the Cell record, ANSI-256 terminal render replaces canvas.
