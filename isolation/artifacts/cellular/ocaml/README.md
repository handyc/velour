# cellular — OCaml port of s3lab Cellular sublab

Phase 3 of the cellular multi-platform port. Hand-port from the
Phase 1 C distillation (`isolation/artifacts/cellular/c/`) into
OCaml 4.14 with stdlib only — no opam deps, no dune required.

## Build + run

```
ocaml cellular.ml                        # interpret directly
ocaml cellular.ml -r 200 -s 42           # 200 rounds, deterministic
ocaml cellular.ml -r 200 -p 0            # no periodic render
```

To compile to a native binary:

```
ocamlfind ocamlopt -linkpkg cellular.ml -o cellular
./cellular -r 200
```

## What you should see

ANSI-256 terminal blocks, 16×16 grid of colored 2-cell tiles, one
per population cell. The script clears the screen + repaints every
25 rounds by default (`-p 0` to disable; `-p N` to render every Nth).

After the configured `-r N` rounds, prints a final render + the
"completed" line on stderr.

## Algorithm parity

PRNG (xorshift32 + Park-Miller LCG), step_grid, fitness, mutate,
palette_inherit, neighbour_idx — all line-for-line ports of the
JS reference in `static/s3lab/js/engine.mjs`. The hex-offset
neighbour tables (DY/DXE/DXO and NB_DC_EVEN/NB_DC_ODD/NB_DR) are
the same constants.

What's different: `Bytes` instead of `Uint8Array`, OCaml's
exception-friendly `getopt` open-coded, ANSI-256 render replaces
the canvas. Population state is an `array` of records.

## Generated by

Hand-port from `isolation/artifacts/cellular/c/cellular_c.c`
(Phase 1, distilled by `condenser/distill_cellular.py`). Not
auto-generated; OCaml's grammar is far enough from C that a
1:1 transpilation would be ugly. Future: a Condenser
`distill_cellular_ocaml` could codify the patterns used here
(record-as-struct, ref-as-mutable-int, Bytes-as-uint8array).
