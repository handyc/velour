# Velour view-decorator audit

Generated by `manage.py audit_view_decorators` — re-run after
any decorator change to keep this current.

## Why this exists

Velour is moving toward a multi-user model with admin users + many
limited-access users. Today's auth model is essentially binary:
a view is either open or `@login_required`. There are exactly
**two** finer-grained gates in the entire codebase (one staff,
one superuser), and **zero** uses of Django's permission system.
This report is the inventory you need to make policy decisions:
which views should stay LOGIN, which should escalate to STAFF or
SUPER, and which need real Django permissions.

## Recommendations for the multi-user transition

1. **Audit the OPEN views first.** Some are intentional (displacer,
   landingpage, ESP API endpoints with token auth). Anything in the
   table above that does NOT match a deliberate "should be public"
   intent should be tightened to `@login_required` or stricter.

2. **Tier the LOGIN views by sensitivity.** 565 views all gated only
   by `is_authenticated` is too coarse for a multi-user deploy.
   Suggested tiers:
   - **System admin (SUPER):** terminal (already done), services,
     security, maintenance, sysinfo, logs, hosts, app_factory,
     databases, winctl, backups
   - **Content editor (STAFF or PERM):** codex, displacer, attic,
     identity (the editorial side), chronos task management
   - **Per-app member (PERM):** most creative apps — give a user
     permission to access aether OR naiad OR studious individually
     so a "limited user" only sees what they have access to
   - **Read-only viewers:** dashboards, briefings, public manuals

3. **Add MFA before any public-IP deploy.** Tiered permissions
   plus weak password authentication is still weak. The `django-
   two-factor-auth` package is the standard answer.

4. **Add login-attempt lockout.** `django-axes`. ~30 min to wire.

## How to keep this current

```
venv/bin/python manage.py audit_view_decorators --update
```

Without `--update`, prints the report to stdout for inspection.
With `--update`, rewrites this file in place.
