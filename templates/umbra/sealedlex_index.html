Sealed linguistic-CSV pipeline. For humanities and area-studies work on low-resource languages — Hindi, Sanskrit, Konso, Amharic, Tigrinya, and others — where each cell is one linguistic form and you want per-character analyses (char-class counts, codepoint matches, length, cell equality) to run sealed. Concrete's programmable bootstrapping gives native byte-level lookup tables under seal, which is exactly the shape per-cell symbolic ops want.

Threat model: key, encrypt, ops, decrypt all in one Django request. Pedagogical demo of the sealed pipeline shape, not yet a real privacy boundary. A two-process split lands later.

Hard caps for in-request runs: {{ max_cells }} cells per op, padded to {{ max_cell_len }} bytes per cell. Each byte burns one PBS call (~200 ms), so the wall fills quickly. Bigger corpora go through manage.py sealedlex_run <slug> (no cap; chunks of {{ max_cells }} reuse the same compiled circuit), and manage.py sealedlex_slurm <slug> generates a Leiden ALICE batch script (defaults 20 CPUs × 4 h).

Language profiles. ASCII covers Latin-script forms; Devanagari covers Hindi / Sanskrit / Marathi; Ge'ez covers Amharic / Tigrinya / Ge'ez. More profiles land per Leiden's research needs.
