Linguistic CSV through TFHE. For humanities work on low-resource languages — Hindi, Sanskrit, Konso, African sign languages, and others — where each cell is one linguistic form and you want per-character analyses (char-class counts, vowel counts, cell lengths) to run sealed. Concrete's programmable bootstrapping gives us native byte-level lookup tables under seal, which is exactly what per-cell symbolic ops need.

Same threat model as the CSV lab: key, encrypt, ops, decrypt all in one Django request. Pedagogical demo of the sealed pipeline shape, not a real privacy boundary.

Hard caps for in-request runs: {{ max_cells }} cells per op, padded to {{ max_cell_len }} bytes per cell. Each byte burns one PBS call (~200 ms), so the wall fills quickly. Bigger corpora belong on ALICE — a manage.py corpuslab_run follow-up will batch via Slurm arrays.

Honest caveat. ASCII bytes only for the moment — Devanagari, Ge'ez, Tifinagh and other non-ASCII scripts read as class other. Codepoint-aware LUTs land in a follow-up.
