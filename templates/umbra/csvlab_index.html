Pedagogical demo. Secret key, encryption, cell ops, and decryption all run in the same Django request — so this shows the round-trip math works, not a real privacy boundary. Every numeric cell becomes its own CKKS ciphertext; ops touch only ciphertexts; the server decrypts at the end.

Hard cap: {{ max_numeric_cells }} numeric cells per CSV. Each cell becomes ~330 KB of ciphertext and takes ~7 ms to encrypt, so anything larger hangs the request. String cells are free — they're never encrypted.
