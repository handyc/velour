How byte attribution works

For each fork the analyser shells out to nm --print-size on the unstripped *.dbg twin (built with the same flags as the shipping binary, just without -s). Each symbol is matched against an ordered prefix/exact pattern table in officelab/analyzer.py — first match wins, so add specific patterns above general ones. Bytes left over after every symbol is attributed (ELF headers, program headers, alignment padding, the .rodata pool of ANSI escape strings, etc.) show up as elf overhead on the per-version page.

If you add a new feature to office7+ and see uncategorized bytes, extend the pattern table and reload — the analyser runs live on every page load (no cache, no DB).
