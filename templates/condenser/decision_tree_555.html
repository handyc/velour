Decision Tree → 555 Timer ICs

The final tier of the Condenser pipeline: binary decision trees implemented entirely in analog hardware. No code, no microcontroller — just comparators, resistors, and capacitors.

ATTiny13a → distill → 555 timers Same decision logic, zero lines of code
Core insight: A 555 timer in comparator mode is a single branch node in a decision tree. It answers one question: is this voltage above or below a threshold? Chain N−1 of them together and you classify an analog signal into N discrete states — exactly like the ATTiny's binary decision tree, but with no instruction set, no clock, no program memory. The logic is the circuit.

The 555 as a comparator

The 555 has two internal comparators fixed at 1/3 VCC and 2/3 VCC. But pin 5 (CONTROL) lets you override the upper threshold with any voltage — set it with a resistor divider and each 555 becomes a voltage comparator with an arbitrary trip point. 

        ┌────────────┐
 GND ───┤1   555    8├─── VCC (5V)
TRIG ───┤2          7├─── DISCH (unused)
 OUT ───┤3          6├─── THRESH ─── tied to TRIG
 RST ───┤4          5├─── CTRL ←── voltage divider sets threshold
        └────────────┘

Comparator mode wiring:
  • Tie pins 2 (TRIGGER) and 6 (THRESHOLD) together → analog input
  • Pin 5 (CONTROL) → set threshold via R1/R2 divider from VCC
  • Pin 4 (RESET) → VCC (or driven by upstream 555 for gating)
  • Pin 3 (OUTPUT) → HIGH when input < threshold, LOW when above
  • Pin 7 (DISCHARGE) → leave floating (unused in comparator mode)

When the input on pins 2+6 is below the CONTROL voltage, the output goes HIGH. When above, it goes LOW. That's one if/else branch — one node of the tree.

4-leaf decision tree (3 × 555)

The simplest useful tree: one root comparator splits the input into two halves, then a second level of two comparators subdivides each half. Four output states from a single analog sensor.

Gating logic:

  555 #1 output HIGH → 555 #2 RESET pin = VCC (enabled)
                     → 555 #3 RESET pin = GND (disabled)

  555 #1 output LOW  → 555 #2 RESET pin = GND (disabled)
                     → 555 #3 RESET pin = VCC (enabled)

  The RESET pin (pin 4) enables/disables each downstream 555.
  When RESET is LOW, the 555 output is forced LOW regardless of input.
  When RESET is HIGH, the 555 operates normally.

  To invert the root output for gating, use a simple NPN transistor
  inverter (1× 2N2222 + 2× resistors) or dedicate another 555 as
  an inverter by tying its TRIGGER to the signal and THRESHOLD to VCC.

Setting arbitrary thresholds

The CONTROL pin (pin 5) accepts any voltage between GND and VCC. Use a resistor divider to set the exact comparison threshold: 

── Threshold setting via CONTROL pin ──

     VCC (5V)
       │
      [R1]   ← upper resistor
       │
       ├──────── pin 5 (CONTROL)
       │
      [R2]   ← lower resistor
       │
      GND

  Threshold = VCC × R2 / (R1 + R2)

  Common thresholds with standard resistor values:

  1.0V  →  R1=390KΩ, R2=100KΩ  (1.02V)
  1.5V  →  R1=220KΩ, R2=100KΩ  (1.56V)
  2.0V  →  R1=150KΩ, R2=100KΩ  (2.00V)
  2.5V  →  R1=100KΩ, R2=100KΩ  (2.50V)
  3.0V  →  R1=100KΩ, R2=150KΩ  (3.00V)
  3.5V  →  R1=100KΩ, R2=220KΩ  (3.44V)
  4.0V  →  R1=100KΩ, R2=390KΩ  (3.98V)

  Use 100KΩ+ values to keep current draw under 50µA per divider.
  Add a 100nF bypass cap from pin 5 to GND for noise immunity.

  H = warm + bright + wet  → "greenhouse"      → green LED

Equivalence to the ATTiny decision tree

The same logic, zero abstraction layers apart:
  • The ATTiny's if (adc_reading < threshold) becomes a 555 comparator
  • The ATTiny's tree array in PROGMEM becomes the physical wiring topology
  • The ATTiny's decide() tree walker becomes signal propagation through the cascade
  • Response time: ATTiny ~100µs (ADC + tree walk) vs. 555 ~10µs (pure analog)
  • Power: ATTiny ~5mA active vs. 555 tree ~1mA per chip (all quiescent)

The 555 tree has no firmware to corrupt, no clock to fail, no registers to overflow. It will keep classifying as long as the power supply holds. The logic is the hardware.

Scaling limits

Practical upper bounds for 555-based decision trees:

  • 4 leaves (3 nodes): Breadboard-friendly. 3 ICs. Runs on a 9V battery for weeks.
  • 8 leaves (7 nodes): Still practical. 4 DIP-14 packages (using 556 duals). ~35mA total.
  • 16 leaves (15 nodes): Pushing it. 8 packages, gating logic gets complex, signal integrity matters. Consider a PCB.
  • 32+ leaves: Use an ATTiny13a instead. You've crossed the threshold where a $0.50 microcontroller with 1KB of flash is simpler, cheaper, and more reliable than the equivalent discrete circuit.
The crossover point: A 555 decision tree beats a microcontroller when you need zero firmware, instant response (~10µs), or absolute reliability in harsh environments (no clock to drift, no registers to corrupt, no code to crash). Above ~16 leaves the wiring complexity exceeds the firmware complexity, and the ATTiny wins on every axis — that's exactly where the Condenser pipeline's ATTiny tier begins.
