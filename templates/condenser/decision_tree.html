Decision Tree → ATTiny13a

Multi-channel ADC decision tree + picoUART TX + Gödel self-reference. Sub-1KB firmware that reads sensors, makes decisions, and can describe its own logic.

ATTiny13a: 1KB flash, 64B SRAM, 10-bit ADC, 6 GPIO.
3 ADC channels with RESET intact (PB2/PB3/PB4). 4th channel (PB5) requires RSTDISBL fuse.
picoUART TX uses ~40 bytes. Decision tree + Gödel self-reference fits in ~800 bytes total.
┌──────────┐ RST ──┤1 T13a 8├── VCC PB3 ──┤2 7├── PB2 (ADC1, ch0) PB4 ──┤3 6├── PB1 (LED or OC0B audio) GND ──┤4 5├── PB0 (TX → ESP) └──────────┘ PB3 = ADC3 (ch1) PB4 = ADC2 (ch2) PB5 = ADC0 (ch3, needs RSTDISBL) PB1 = OC0B (Timer0 PWM — bytebeat audio output)
Gödel self-reference: The chip stores a compressed description of its own decision tree in flash. Ground all sensor inputs (the "who are you?" gesture) and it transmits its self-description + a CRC-8 of its own program memory. The ESP can reconstruct the entire tree without the source code. The code IS its own documentation.

Bytebeat: PB1 becomes a PWM audio output. Each decision tree leaf value selects a bytebeat formula — the chip plays different music depending on what it senses. Overrides the LED. Do not connect a speaker directly — PB1 can only source ~40mA. Use one of the three wiring options below.
Formulas: sierpinski · crowd · acid · bells · drums · underwater · alarm
Wiring diagrams (3 options) 
── Option A: Piezo buzzer (simplest, no extra parts) ───────────

  Piezo elements are high impedance — safe to drive directly.
  Tinny sound, but audible. Good for alarms and notification tones.

  Parts: 1× passive piezo buzzer (not active/driven type)


── Option B: RC low-pass → ESP ADC or line-in (quiet, clean) ──

  Smooths the PWM into analog audio. Feed into the ESP's ADC
  for digital capture, or into a powered speaker's line/aux input.
  The 1K + 100nF filter gives a ~1.6kHz cutoff — removes the
  37.5kHz PWM carrier, keeps the 0-8kHz audio content.

                 ┌──────────┐
  sensor ──PB2 ──┤7  T13a  8├── VCC
  sensor ──PB3 ──┤2        7├──
  sensor ──PB4 ──┤3        6├── PB1 (OC0B)
           GND ──┤4        5├── PB0 (TX)
                 └──────────┘       │
                                    │ (TX → ESP RX for UART data)
                  PB1 (OC0B)        │
                      │             │
                      ├──[1KΩ]──┬───────── audio out ──→ ESP ADC
                      │         │                        or line-in
                      │       ──┴──
                      │       ──┬── 100nF
                      │         │   (ceramic cap)
                      │         │
                      GND ──────┘

  Parts: 1× 1KΩ resistor, 1× 100nF ceramic capacitor
  Note: For headphones, add a 10µF coupling cap in series
        with audio out to block the DC offset:

        audio out ──┤├──[10µF]──→ headphone tip
                                  headphone sleeve → GND


── Option C: NPN transistor amp → 8Ω speaker (loud) ───────────

  Class A amplifier using a single 2N2222 or BC547. Drives an
  8Ω speaker at ~100mW — audible across a room. The 10µF cap
  blocks DC so the speaker only sees the audio signal.
