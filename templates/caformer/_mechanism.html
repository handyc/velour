how it works: {% if slug == 'embedding' %} Seed a 16×16 grid from (token_id, position) via an LCG so identical tokens at identical positions produce identical grids. Step 4 ticks of rule_embed. The 256-cell K=4 result is the 512-bit embedding vector that downstream attention consumes. {% elif slug == 'layer_norm' %} rule_norm is evolved toward a "minority-colour attractor": each tick, cells tend to flip toward the locally-rare colour. Run for 4 ticks and the colour histogram flattens — the CA-analogue of mean-shift + rescale. Used pre-attention and pre-MLP every block. {% elif slug == 'self_attention' %} For each (Q, K) token pair, stack the two 16×16 grids vertically (a 32×16 stripe), step 1 tick of rule_score, count cells where colour < 2 in the top half — that integer is the attention weight. rule_mix then blends the chosen V grid into the state. Causal mask is enforced by only scoring j ≤ i. {% elif slug == 'projection' %} The Q/K/V projection is one CA tick of the rule named after it (this view shows rule_q; the K and V rules have the same shape). Running on the same input grid produces three different "views" — Q says what am I asking, K says what do I offer, V says what would I return. This is the cell-level analogue of W_Q · x, W_K · x, W_V · x. {% elif slug == 'mlp' %} Tile the 16×16 input 2×2 → a 32×32 grid (the wider "hidden dim"), step 2 ticks of rule_mlp on that expanded grid, then majority-pool each 2×2 block back down to 16×16. The rule's non-linear lookup plays GELU; the expand/contract plays W_1 / W_2. {% elif slug == 'transformer' %} One block = pre-norm → self-attention → residual_merge → pre-norm → MLP → residual_merge. This view shows the rule_mix step (the attention readout); the other 6 block rules each contribute a sub-stage. Stack N blocks for nano_gpt / GPT-2 / GPT-3-shape. {% elif slug == 'softmax' %} The sampler reads the output grid's 4 cell counts, treats them as un-normalised logits over 4 vocab buckets (each byte v maps to bucket v >> 6), divides by temperature, picks proportionally. Temperature noise is itself sourced from a CA byte stream (so identical seeds → identical samples). {% elif slug == 'output' %} On the last token's CA state, run 2 ticks of rule_output, count cells per colour, map each vocab byte v to its bucket (v >> 6 → one of 4 buckets), assign that bucket's count as the byte's logit. Sampler reads those 256 logits. This is the LM head. {% else %} For every cell, read the 7-cell hex neighbourhood, pack into the 14-bit key, look up the rule's 2-bit output, write it to the next grid. {% endif %}

readout: {% if slug == 'embedding' %} The grid is the embedding: the bar histogram on the left is its 4-colour "distribution". Position-embed XOR's in next; then the whole block consumes it as token state. {% elif slug == 'layer_norm' %} A flatter histogram = stronger normalisation. Adjacent stages (attention, MLP) read the normed grid as their input. {% elif slug == 'self_attention' %} The output is the attended CA grid — V[best_j] XOR state[i], rule_mix'd by one tick. This grid then feeds into residual_merge with the original state. {% elif slug == 'projection' %} The output grid is one of Q, K, V for this token. Q and K enter the per-pair attention scorer; V is the payload that gets pulled in proportionally to that score. {% elif slug == 'mlp' %} Output replaces the token's state via a residual_merge. In a well-evolved MLP the cell-change rate is high (the rule is mixing information) without colour collapse (it still uses all 4 colours). {% elif slug == 'transformer' %} After both residuals, the per-token grid is the input to the next block — or, after the last block, to the output head. {% elif slug == 'softmax' %} The histogram bars are the 4 logit buckets — the sampler picks a byte proportionally to its bucket's height (with temperature). {% elif slug == 'output' %} The histogram is the un-normalised logits over the 4 vocab buckets that cover all 256 bytes. Sampler is one softmax+CDF step away from emitting the next token. {% else %} The output grid is the activation that the next stage of the forward pass consumes. {% endif %}
