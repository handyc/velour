Pipeline diagram

How ca_forward_qkv wires the 8 primitives into a full transformer-shaped forward pass using 10 evolvable 16,384-byte rule tables. Every arrow is one or more hex_ca_step calls — there is no float math, no learned weights in the conventional sense, only K=4 hex cellular-automaton rules the GA evolves.

10 evolvable rule tables — what each does

ruleused byrole
embed ca_embedding + ca_positional_embedding Token id (and position) → 16×16 CA seed grid via N ticks of this rule.
qca_qkv_project Builds the per-token "query" CA grid for attention.
kca_qkv_project Builds the per-token "key" CA grid.
vca_qkv_project Builds the per-token "value" CA grid.
scoreca_attention_score Reduces a Q-K pair to a scalar attention weight via N CA ticks.
mixca_self_attention Combines weighted V grids into the attended CA grid (the readout).
mergeca_residual_merge Skip-connection: one CA tick blending sub-block output back into the input.
normca_layer_norm_iterative Pre-norm + final norm. Defaults to a "minority-colour" rule (analogue of mean-shift).
mlpca_mlp Per-token feed-forward: 2 ticks of this rule on a 2× expanded grid, then crop.
outputca_output_head_iterative Final CA on the last token; cell-counts of each colour build the V-dim logit vector.

Composition ladder — wrapping this same forward pass

L1  primitive          one of the 8 ops above (single CA + rule)

L2  nano_gpt           ca_forward_qkv(n_blocks=3)
                       embed → 3 × block → norm → output

L3  chat_gpt2_shape    ca_forward_qkv(n_blocks=12)
                       "4 nano_gpts in series"

L4  chat_gpt3_shape    n_branches × chat_gpt2_shape (parallel branches,
                       sum or vote blend)

L5  chat_gpt3_5_shape  cheap CA router → top-k of n experts (each = L4)
    chat_gpt3_5_hybrid same MoE, but experts can be CA OR a real LLM
                       (e.g. distilgpt2). The router is always CA;
                       only the expert can be LLM.

L6  default_mode_loop  no input — pipe the model's own output back
                       into its input via a continuous CA tick loop.

Where evolution happens

Every box above with a _rule label is a 16,384-byte lookup table the GA can mutate. A genome is the tuple of all 10 tables = 163,840 bytes. Fitness is mean log-probability of the corpus next byte (or one of the role-specific fitnesses on the component page).
