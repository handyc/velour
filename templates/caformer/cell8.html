cell8 · 8-cell-input hex CA primitive

The default caformer CAs read 7 cells (self + 6 hex neighbours) and produce 1 output cell — LUT size 47 = 16,384 entries. The cell8 primitive adds an eighth input cell: an external input port that any signal can be wired into. LUT size becomes 48 = 65,536 entries.

What the input port is for

The input port carries one K=4 value per cell per tick, sourced from outside the local hex neighbourhood. Use cases:

  • Chain composition: wire rule B's input port to rule A's output cell. Rule B's behaviour now depends on what rule A emitted — first-class cross-chain signalling without neighbourhood propagation.
  • Global modulation: broadcast a single bit (mood, DMN heartbeat phase, sensor reading) across every cell's input port. Same rule, different macro behaviour per signal.
  • Attention primitives: a "query" cell selects which of K source cells flows in. This is the CA-native sibling of the transformer's gated attention.

How the LUT key is laid out

Bit packing (high → low, 2 bits per cell, K=4):

key  = input_port[14..15] | self[12..13]   | nw[10..11]   | ne[8..9]
       | r[6..7]            | se[4..5]      | sw[2..3]     | l[0..1]

Total: 16 bits → 65,536 distinct keys → 65,536 LUT entries.
Each entry: 2 bits = the next-tick colour for the centre cell.

When input_port is held at 0 across all cells, the high 2 bits of the key never vary. The rule then effectively uses only 1/4 of its LUT (the entries with key >> 14 == 0), reducing to 16,384 active entries — exactly equivalent to a 7→1 rule. Adding signal on the input port unlocks the other three quadrants.
