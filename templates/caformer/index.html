caformer — a transformer made of cellular automata

Workshop for replacing every component of an LLM with a network of CAs. The 8 primitives below are the layers any transformer needs; the compositions below stack and parallelise them all the way up to a GPT-3-shaped model. The plan is to evolve each level against an appropriate fitness, then compose the winners.

Strategy — compress LLMs while replacing components with CAs

  1. Primitive replacement. Try replacing each transformer op with a single class-4 CA. If single-CA doesn't preserve the op's behaviour, escalate to a chain or network of CAs. Single-CA stand-ins shipped: caformer.primitives.ca_softmax_sample, ca_mlp, ca_layer_norm.
  2. Evolve role-by-role. Each component has a different fitness — denoising, distillation, normalisation, attention, sampling — and gets its own GA. See spoeqi chain GA with its 17 role-aligned metrics for the template.
  3. Stack into nanoGPT. Composition L2 — three transformer blocks in series + embedding + output head. Same evolution loop, longer sequence.
  4. Recursive scale-up. Wrap in series + parallel for GPT-2 → XL → 3 → 3.5-shapes. The previous level's evolved outputs become the next level's building blocks.
  5. CA-generated training data. Replace data vectors themselves with deterministic CA streams — a Pact + domain becomes a "dataset of N tokens". Two researchers with the same Pact instantly hold identical training corpora; no dataset shipping, no licensing surface.
  6. Default-Mode loop (L6). Once the stack works, wrap it in a no-input feedback loop — CA → text → tinyLLM → text → CA — so the model "thinks" continuously without external prompts. See caformer/dmn.py:dmn_loop.
