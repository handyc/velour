quarter compose · cell8

A cell8 rule (65,536-byte LUT) is structurally four stacked 7→1 rules (16,384 bytes each), one per port value. Pick four trained QRPairs, the system slices out each pair's rule at the chosen position, stacks them into one cell8 LUT, then runs CA inference at all 4 port values on a 256×256 board. Same LUT, different port, different output. Demonstrates pool-as-personality without any new training.

what this proves

  • Mechanism: port value selects which of 4 stacked rules fires. If all 4 quarters were the same rule (special case), all 4 port outputs would be identical — that's the upcast_7to1_to_cell8 case.
  • Diversity count: shows how many distinct bytes the 4 ports produced. 4 = full personality switching; 1 = port-agnostic (or rules happen to converge).
  • Why bytes may not match target chars: the underlying 7→1 rules were trained on a 128×128 board with different geometry; running them as cell8 quarters on a 256×256 board produces stable but different bytes. Real production use requires retraining 7→1 rules at board256 first (the ALICE bundle pipeline handles this).
