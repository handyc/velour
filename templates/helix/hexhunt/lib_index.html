hexhunter — standalone library

The class-4 hex-CA hunter, repackaged as a reusable library. Pure algorithm: no DNA corpus, no Velour models, no I/O. Drop the source into any project; same behaviour from C, Python, or JavaScript thanks to a deterministic 32-bit LCG threaded through every call.

 →

Upload an existing ruleset; the GA continues around it and returns the new best.
→

assemble — 16 × 4096 B → 64 KiB

Drop or evolve 16 rulesets into a 4×4 grid; concatenate them in slot order into a single 65,536-byte file. Useful for fleet-style packets, ESP firmware tails, or any consumer that wants a fixed 64 KiB region of rule data.
