algorithm

The hexhunter searches the space of 4-state hex cellular automata for class-4 dynamics — rules that are neither flat (class 1/2) nor noisy (class 3), but live on the edge-of-chaos band where structures form, drift, and interact. The search is a small, deliberately stupid genetic algorithm; the interesting part is the fitness.

genome

A hex CA cell has 6 neighbours plus itself — 7 cells, each in one of K=4 states. That's K⁷ = 16,384 distinct local situations, and a complete rule maps each one to the next colour of the centre cell. Each entry needs only 2 bits, so the rule packs into: 

K⁷ situations × 2 bits / 8 = 4096 bytes

The packing convention is (self << 12) | (n0 << 10) | ... | n5, with neighbours n0..n5 in row-parity-sensitive offset order. The same indexing scheme is used by the main automaton app, by spoeqi, and by every port of this library.

identity seed

Every run starts from the identity rule — every situation maps the centre to its current colour. This gives the GA a flat zero-activity floor to climb from: 

genome[0..1024)    = 0x00   // self=0 → next=0
genome[1024..2048) = 0x55   // self=1 → next=1
genome[2048..3072) = 0xAA   // self=2 → next=2
genome[3072..4096) = 0xFF   // self=3 → next=3

fitness — smooth tent on activity

For each candidate genome, run a deterministic 14×14 hex grid for HORIZON=25 ticks and look at the last third of its activity history (fraction of cells that flipped per tick). Score is the sum of:

  • +1.0 if the final grid is not uniform
  • +1.5 if the activity tail never fell to zero (aperiodic proxy)
  • +2.0×activity_reward — smooth tent peaking at activity=0.12
  • +0.25×diversity if ≥2 colours each occupy ≥1% of the final grid

The activity reward is the load-bearing piece:

if avg ≤ 0.12: reward = avg / 0.12          // 0 → 1
else:           reward = (0.75 - avg) / 0.63  // 1 → 0 at avg=0.75

No hard cutoff: a near-dead genome with avg=0.005 still gets a tiny non-zero pull toward the band, so the GA always has a gradient to climb. The original cutoff version stalled at fitness ~3.5 because every barely-interesting genome scored identically.

GA

Trivially small: tournament breeding from the top half of a sorted population. 

1. seed pool[0] = identity, pool[1..POP-1] = mutants of identity at init_rate (0.05)
2. for GENS generations:
     - score every genome on the same grid_seed
     - sort descending by fitness
     - bottom half ← cross(top[a], top[b]) then mutate at breed_rate (0.005)
3. final sort, return pool[0]

Selection is implicit (top half always survives), crossover is one-point random, mutation flips each of the 16384 entries with probability rate to a new uniform colour. Defaults are POP=30, GENS=40, the same as the original ./hunter program.

determinism

All randomness comes from a single Park-Miller LCG (state*1103515245 + 12345) seeded from rng_seed. Two runs with the same configuration produce the same 4096-byte ruleset — in C, in Python, in JavaScript. The library was rebuilt from the original oneclick_class4/hunter.c specifically to remove the libc rand() dependency that broke this guarantee across platforms.
