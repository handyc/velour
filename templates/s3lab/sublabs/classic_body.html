In-browser bench for the ESP32-S3 SuperMini + ST7735S hex-CA + GPIO setup. Hunt a class-4 ruleset, wire cells to virtual GPIO pins, and download genome.bin + gpio_map.txt ready to drop straight onto a real device — same algorithm, same file formats, no flashing required for design iteration.

Default layout: inputs on a 2×2 grid in the interior re-seed central cells; outputs sample both vertical edges and fire when a state-3 wave reaches the edge. Toggle inputs HIGH/LOW to inject stimulus, watch propagation in the timing diagram. The default fills the 9 free GPIOs the SuperMini leaves after the TFT (5 outputs + 4 inputs) — anything you design here is physically buildable; add more pins only if you switch to a module with more breakouts.

      The algorithm here is a line-for-line port of isolation/artifacts/hex_ca_class4/esp32_s3_full/src/main.cpp: same K=4 hex CA, same 16×16 grid, same 4096-byte packed genome, same class-4 fitness, same 4-bit output state-mask, same INPUT_PULLUP input semantics. genome.bin uses the 4104-byte HXC4 ‖ palette ‖ genome tail format from the standalone hunter — files round-trip both directions with no conversion. The hunt runs in a Web Worker so the UI stays responsive.

      Pin map on the SuperMini: TFT uses 4–7 + 11–12; USB D-/D+ is on 19/20; BOOT on 0. The 9 free pins are 1, 2, 3, 8, 9, 10, 13, 14, 21 — the lab default uses all of them (outputs: 1, 2, 3, 8, 9; inputs: 10, 13, 14, 21). Typical "logic processor" wiring is 4 inputs + 4 outputs, which leaves one pin spare.
