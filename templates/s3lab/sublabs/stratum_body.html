{# Stratum sublab — two-layer hex CA: library + meta-CA. #}

Two coupled layers of the same K=64 hex CA, evolving together. The library on the left is 64 distinct K=64 HexNN rules, each with its own ruleset, palette, and running grid; tournament rounds replace underperformers with mutated copies of winners every few seconds. The meta-CA on the right is a 16×16 hex grid whose cell states name library entries — a state of 5 means "show library #5 here." The meta-CA steps using the elite library entry's ruleset, so as evolution shifts who's at the top, the meta-CA's stepping rule changes with it. Watch one rule playing two scales of itself simultaneously.

The library doubles as the meta-CA's palette: instead of mapping a state to a colour, we map it to a whole running CA. Multiple meta-cells with the same state show identical patterns — which makes regions of the meta-CA visible at a glance. Gold borders highlight the elite (the library entry currently driving the meta-CA's stepping). Click any tile on either canvas to download its genome as a hexnn-genome-v1 JSON file (round-trips into the /hexnn/ page unchanged).

HexNN engine extracted from the /hexnn/ bench into static/s3lab/js/hexnn_engine.mjs. Same K-dialable hex CA, same mulberry32 PRNG, same nearest-prototype lookup, same edge-of-chaos parabola fitness on the K=4-quantized change rate — just modular instead of inline. K=64 is dialable up to K=256 if you want to push the rule expressivity further (recompile the constant in stratum.mjs).

Each tournament round picks two random library entries, scores them on a shared seed, replaces the loser with a mutated copy of the winner. Loser inherits the winner's palette outright — over several rounds dominant rules visibly share colour schemes, which is how you see family resemblance forming. The elite is whoever has the highest fitness in the current library; if a tournament discovers a better entry the meta-CA's stepping rule changes immediately.

Cost budget at default settings: 64 inner CAs × ~64 µs each ≈ 4 ms per inner tick; one meta-CA step = ~50 µs; one tournament round ≈ 1 ms (two 25-step fitness evals). The dominant cost is the render — 64 library thumbnails + 256 meta-cells, each painting a 16×16 inner grid via the shared ImageData buffer.
