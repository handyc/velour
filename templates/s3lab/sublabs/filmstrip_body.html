{# Filmstrip sublab — strip of past CA refinements scrolling left. #}

A scrolling strip of recent CA refinements. The live tile on the right keeps stepping; every time a refine completes, the strip shifts left and the just-superseded genome becomes a new frame in the history. By default older frames are frozen (drawn once, zero CPU); flip run history on to keep them all stepping in parallel.

Same engine as Classic — K=4 hex CA, 16×16 grid, 4096-byte packed genome, 30-agent / 40-gen GA hunt in a Web Worker. The only thing that changes is what you do with the post-refine state: Classic discards the old genome and runs the new one in-place; Filmstrip pushes the old genome onto a ring buffer of the last 7 refinements and shows them all at once. Useful for seeing how a warm-start refine drift relates to the original rule, and for spotting visual lineages over many auto-refine cycles.

No GPIO, no timing diagram, no upload — Classic owns those. Switch tabs at any time; both sublabs share the same engine code under static/s3lab/js/engine.mjs / hunt.mjs / worker.mjs but keep their own genome state, so flipping between tabs starts each one fresh.
